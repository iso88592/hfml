About the syntax

HFML is designed to use the least amount of resources possible to render complex web pages, therefore we use the least amount of tokens possible in parsing all the language primitives. There are a few rules to understand:

  • Everything insde <> is considered and rendered as text.
  • Text can be modified to have special semantic attributes with []. This is equivalent to HTML's elements like tags, classes, ids, etc.
  • There are predefined attributes that function as HTML's tags. They are usually the same as the original HTML tags' name.
  • Elements starting with a # are considered to be ids.
  • Attributes can have general and special modifiers that are separated by :. Any number of modifiers are accepted.
  • Events are special attributes. There is no need for any client side programming, since every event can be handled the predefined event handlers.
